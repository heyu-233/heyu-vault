# SIG-Kernel

## What To Track

- SIG maintainers and communication channels
- open issues and labels
- accepted patch style
- branch policy
- test and review expectations

## Questions Before Contribution

- Is the issue still available?
- Which branch should the PR target?
- What tests are mandatory?
- Is there a reproducer or only a requirement description?
