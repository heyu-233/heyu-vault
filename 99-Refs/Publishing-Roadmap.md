# Publishing Roadmap

This vault does not publish a full digital garden yet. The first goal is to keep Markdown readable on GitHub and usable in Obsidian.

## Options

| Option | Use When | Notes |
| --- | --- | --- |
| GitHub Markdown | Default now | No build step, easiest to maintain |
| MkDocs | Need a documentation site | Good for structured technical docs |
| Quartz | Need a digital garden | Good Obsidian-style publishing path |
| Obsidian Publish | Need managed hosting | Paid, low maintenance |

## Current Default

Use GitHub Markdown as the public reading layer.

## Before Publishing More

- Review [[../PRIVATE_POLICY|Private Policy]].
- Keep `Daily/` and `Inbox/` out of public summaries unless cleaned.
- Prefer publishing mature notes from `MOCs/`, concepts, labs, source reading, and cheat sheets.
